% ── What it is ───────────────────────────────────────────────────────────────
\vspace{6pt}
{\large\bfseries Second Opinion} streams a live differential diagnosis — not as a chat response, but as a structured reasoning trace you can watch unfold in real time.

% ── The problem ──────────────────────────────────────────────────────────────
\section{The Problem}

Most AI tools in medicine return a diagnosis. A diagnosis without explanation is useless to a student and indefensible to a clinician. When an experienced physician evaluates a patient, they don't jump to a conclusion — they build a list of possibilities, weigh the evidence, revise their thinking in real time, and ask the right questions. That process has never been made visible. Until now.

% ── What it does ─────────────────────────────────────────────────────────────
\section{What It Does}

Describe a patient presentation in free text. The model streams a live differential — 4--6 hypotheses with probability estimates that update as evidence arrives, supporting and refuting findings surfaced per hypothesis, recommended investigations, a conclusion, and clarifying questions that would most narrow the differential. Answer the questions, rerun — watch the differential update.

Every step is visible as it happens. The model cannot output a bare answer — it must justify each decision, one structured event at a time.

% ── How it works ─────────────────────────────────────────────────────────────
\section{How It Works Technically}

The system prompt instructs the model to respond exclusively in NDJSON — one typed JSON object per line, nothing else. Seven event types are defined: \texttt{thought}, \texttt{hypothesis}, \texttt{update}, \texttt{evidence}, \texttt{tests}, \texttt{conclusion}, and \texttt{question}. The server streams these to the client over Server-Sent Events. The frontend parses each line the moment it arrives — hypothesis cards appear, probability bars animate, evidence tags attach — all in real time.

The prompt acts as a schema enforcer. The model is not asked to ``think about'' a diagnosis — it is given a strict output contract with required field names, sequencing rules, and cardinality constraints. Malformed lines are silently discarded server-side. A language model becomes a structured reasoning engine.

% ── Model comparison ─────────────────────────────────────────────────────────
\section{Model Comparison}

Two models stream in parallel, each building its own differential independently. When both complete, results sit side by side — same patient, same moment, different reasoning paths. Currently supporting MiniMax M2, Gemini 2.0 Flash, DeepSeek R1, Qwen3 235B, and GPT-4o.

% ── Tech stack ───────────────────────────────────────────────────────────────
\section{Tech Stack}

Node.js \textperiodcentered\ Express \textperiodcentered\ Server-Sent Events \textperiodcentered\ OpenRouter \textperiodcentered\ Puppeteer \textperiodcentered\ Vanilla JS — no framework, no build step.
